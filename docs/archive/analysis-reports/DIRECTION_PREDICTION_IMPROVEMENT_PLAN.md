# 다음날 방향 예측 개선 및 수정 계획

> 짝 문서: [`DIRECTION_PREDICTION_ANALYSIS.md`](DIRECTION_PREDICTION_ANALYSIS.md) (현상 진단).
> 이 문서는 그 진단을 토대로 **무엇을, 어디를, 어떻게 고칠지**를 우선순위로 정리한다.
> 코드 위치는 작성 시점(`main` 기준) 기준이며, 착수 전 재확인 필요.

## 전제 — 무엇을 목표로 하는가

분석 문서의 결론을 그대로 계승한다:

- **목표가 아닌 것:** "상승확률 숫자 올리기", "고적중률 약속". 화면의 90%대를 그대로 키우는 것은 검증되지 않은 과신을 키우는 것이며 가드레일 위반이다.
- **목표인 것:** "50%를 못 넘는 변별력 → 미세하지만 안정적인 +엣지" + **정직한 확신도 표기**.
- 다음날 단일종목 절대 방향은 구조적으로 ≈50%다. 그래서 전략은 **"하나로 합치기"가 아니라 "다른 질문 두 개로 분리"** 하는 것이다.

### 전략: 절대 방향 + 횡단면 랭킹을 둘 다, 각자의 검증된 신뢰도대로

| | 절대 방향 (`predicted_return` / `up_probability`) | 횡단면 랭킹 (`rank_ic`) |
|---|---|---|
| 답하는 질문 | "이 종목, 들어갈 만한가 / 기대수익 크기는?" | "후보 중 무엇부터 사지 / 자금 배분은?" |
| 검증된 신뢰도 | 방향 AUC 0.508 ≈ 무작위 → **불확실성 정직 표기 필수** | rank_ic 0.092 = 약한 +엣지 → **선택 레이어로 활용** |
| 정책상 위치 | `AGENTS.md`상 매수/매도/보유 **앵커(필수 유지)** | 앵커를 보조하는 우선순위 레이어 |

> 두 경로는 이미 코드에 모두 존재한다(아래 P2 참조). 신규 아키텍처 구축이 아니라 **비중·표기 정합화**가 핵심 작업이다.

## 개선·수정 항목 (영향 큰 순)

각 항목: **문제 → 수정 → 대상 → 검증 게이트 → 기대·한계.**
검증 게이트는 모두 `result/latest/pipeline_report.json` 또는 모델 아티팩트에서 재현 가능한 수치다.

### P1. 수급 피처 부활 (단일 최대 레버 · 두 경로 동시 개선)

- **문제:** `src/data/investor_context.py:51-61`의 `_fetch_flow`가 빈 스텁이라 `foreign_net_buy / institution_net_buy` 및 파생 수급 피처 16개가 전부 상수 0, feature importance도 0. 한국 단기 예측력 있는 신호원 한 묶음이 통째로 죽어 있음.
- **수정:** `_fetch_flow`를 pykrx/KRX 기반 실제 수집으로 구현. 결측/장애 시 빈 DF + `status` 플래그로 안전 폴백(기존 coverage 계약 유지).
- **대상:** `src/data/investor_context.py` (`_fetch_flow`), 의존 피처 계산부.
- **상세 계획:** [`docs/archive/INVESTOR_FLOW_FEATURE_REVIVAL_PLAN.md`](archive/INVESTOR_FLOW_FEATURE_REVIVAL_PLAN.md) 를 1차 근거로 사용.
- **검증 게이트:**
  - 부활 피처의 feature importance > 0 (현재 0).
  - walk-forward **AUC > 0.508**, **rank_ic > 0.092** 중 최소 하나 유의 개선(베이스라인 대비).
  - 누수 점검: 수급 데이터는 **거래일 마감 확정값**만 사용(당일 진행 중 값 금지).
- **기대·한계:** 두 경로 모두에 입력되는 피처라 절대·랭킹 동시 개선 가능. 단 개선 폭은 데이터 품질·커버리지에 좌우되며 "방향 적중률 급등"은 기대하지 않는다.
- **체크리스트:**
  - [ ] pykrx/KRX 수급 수집 함수 구현 + 결측 폴백
  - [ ] 마감 확정값만 사용하는 leak 가드 + 테스트
  - [ ] 부활 후 importance / AUC / rank_ic 재측정, before/after 기록

### P2. 횡단면 랭킹을 1급 출력으로 승격

- **문제:** 유일하게 검증된 엣지(rank_ic 0.092)가 `src/validation/signal_tuning.py`에 살아 있으나, 출력/표시는 종목별 절대 `up_probability`(90%대)가 전면이라 사용자가 "랭커"가 아닌 "방향예측기"로 오해.
- **수정:** 같은 날 종목들의 **상대 순위/백분위**를 출력의 1급 필드로 노출(예: `cross_section_rank`, `rank_percentile`). 절대 신호는 유지하되 부차 표기.
- **대상:** `src/validation/signal_tuning.py`(rank_ic 산출), `src/pipeline.py:675`(튜닝 신호 블렌드), 표시 단계 `src/pipeline.py:905-909`.
- **검증 게이트:**
  - 출력에 일자별 횡단면 순위 필드 존재 + 결정론적(동일 입력 → 동일 순위).
  - 랭킹 상위 분위 vs 하위 분위의 실현수익 스프레드가 양(+) 방향(소규모 universe 한계 명시).
- **기대·한계:** universe가 클수록(수십~수백 종목) 이점 증가. 5종목 같은 소규모에서는 효과가 약하고 노이즈가 큼 — 표기에 이 한계를 같이 노출.
- **체크리스트:**
  - [ ] 횡단면 순위/백분위 필드 산출 및 출력 스키마 추가
  - [ ] 결정론·leak-safe 테스트
  - [ ] 상·하위 분위 스프레드 리포트 추가

### P3. 캘리브레이션 수축 + 정칙화 (과신 교정 · 정직한 확신도)

- **문제:** ① `reg_alpha=0, reg_lambda=0`(`src/models/lgbm_heads.py:95-105`)로 정규화 부재 → 강추세 입력 과신. ② 보정 표본의 92%가 0.4~0.5에 몰려 0.9~1.0 구간 표본이 ≈1개라, 현 5종목이 희소 극단 꼬리로 들어가 보정 신뢰도 낮음(ECE 0.098, Brier 0.262). 보정이 오히려 +1~3%p 상향.
- **수정:**
  - 정칙화: `reg_alpha / reg_lambda > 0` 탐색(과신 억제). config 노출.
  - 꼬리 수축: 보정 표본이 희소한 극단 구간 예측은 **0.5 쪽으로 shrinkage**, 또는 분위수 헤드(이미 `[0.1,0.5,0.9]` 존재)로 **신뢰구간 동반 표기**.
- **대상:** `src/models/lgbm_heads.py`(reg_alpha/lambda), 보정 적용부 `src/pipeline.py:643,650,774`, 표시부 `src/pipeline.py:905-909`.
- **검증 게이트:**
  - **ECE 하락**(< 0.098), **Brier 하락**(< 0.262).
  - 희소 꼬리 종목의 표시 확률이 0.5 쪽으로 이동하거나 신뢰구간 폭이 함께 노출.
  - AUC가 의미 있게 하락하지 않을 것(정칙화가 변별력을 깎지 않는지 확인).
- **기대·한계:** 적중률을 올리지는 않지만 **표시 신뢰도를 정직하게** 만든다. 사용자 신뢰 측면에서 핵심.
- **체크리스트:**
  - [ ] reg_alpha/lambda config화 + 소규모 탐색
  - [ ] 희소 꼬리 shrinkage 또는 분위수 신뢰구간 표기
  - [ ] ECE/Brier/AUC before·after 기록

### P4. 뉴스/공시 계량 피처 활용 (leak-safe)

- **문제:** 정책상 계산 입력으로 허용되나 현재 모델 입력이 아니라 표시/후처리 성격(`investor_event_score`, `jongbae_score` 등).
- **수정:** `src.news_impact` 산출을 **결정론적·감사가능·누수 없는** 피처로 모델 입력화. 한국 뉴스 우선, 발표시각 기준 누수 차단, 결측/429 폴백, OOF 검증 동반.
- **대상:** `src/news_impact/*`, 피처 결합 단계, `src/features/feature_selection.py`.
- **검증 게이트:**
  - 발표시각 이후 정보만 사용함을 보이는 누수 테스트.
  - OOF 기준 AUC 또는 rank_ic 개선(없으면 채택 보류).
  - 동일 입력 → 동일 스코어(결정론).
- **기대·한계:** 단기 뉴스 반영은 변동성이 크다. 누수 차단을 못 지키면 **검증에서만 좋아 보이는 함정**이 되므로 게이트 통과 전 모델 입력 승격 금지.
- **체크리스트:**
  - [ ] leak-safe 뉴스 피처 산출 + 누수 테스트
  - [ ] OOF AUC/rank_ic 게이트 통과 시에만 입력 승격

### P5. 예측 지평선 확대 (구조적, 별도 트랙)

- **문제:** 1일 고정이 가장 어려운 지평선(준효율 + 노이즈 지배).
- **수정:** 주간/월간 지평선 타깃 추가(드리프트가 노이즈 위로 누적). 기존 1일과 병행 제공 옵션.
- **대상:** 타깃 생성(`target_up`/`target_return` 정의부), 학습/검증 구성, 설정.
- **검증 게이트:** 동일 데이터에서 지평선별 AUC/IC 비교표 — 주간/월간이 1일 대비 개선되는지.
- **기대·한계:** "다음날" 제품 정의를 바꾸는 결정이라 제품 합의 필요. 본 계획에서는 **타당성 검증(소규모 비교)** 까지만.

### P6. 레짐 조건부 모델 (후순위)

- **문제:** 추세/변동성 레짐에 따라 신호 유효성이 달라짐.
- **수정:** 레짐 라벨링 후 분리 학습 또는 레짐 피처 추가.
- **검증 게이트:** 레짐별 분리 시 합산 AUC/IC가 단일 모델 대비 개선되는지.
- **기대·한계:** 데이터 분할로 표본이 줄어 과적합 위험. P1~P3 이후에 착수.

## 출력/표시 정합성 — 두 신호가 어긋날 때

절대 방향과 랭킹이 충돌할 수 있다(예: `up_probability` 94%인데 횡단면 4/5위). 이는 버그가 아니라 **정보**다("다 같이 오르는 국면인데 이 종목은 상대적으로 약함" = 유포리아 과신 필터).

- 표시에서 두 값을 **다른 질문의 답**으로 분리해 제시(같은 줄에 섞어 단일 점수로 합치지 말 것).
- 충돌 시 사용자에게 해석 힌트 노출(절대 신뢰도 낮음 + 상대 순위 하위).

어떤 변경도 아래로 판정한다. 게이트는 `pipeline_report.json`의 `walk_forward`, `baselines`, `oof_diagnostics`, `tuned_signal`, `probability_calibration`에서 재현한다.

- before/after 수치를 같은 run에서 기록해 PR에 첨부.

## 비범위 / 하지 말 것

- ❌ 표시 `up_probability` 숫자만 상향(검증 없는 과신 강화).
- ❌ 모델 출력을 내러티브로 임의 오버라이드(가드레일).
- ❌ 누수 게이트 미통과 피처를 "검증에서 좋아 보인다"는 이유로 입력 승격.

## 권장 착수 순서

1. **P1 (수급 부활)** — 두 경로 동시 개선, 가성비 최고.
2. **P3 (정칙화·보정 수축)** — 정직한 확신도, 비교적 국소 변경.
