# 현재 코드 기준, "진짜 필요한" 개선안 우선순위

이 문서는 현재 구현 상태를 기준으로, 투자자 관점에서 효과 대비 비용이 큰 항목부터 정리한 실행 우선순위입니다.

## P0 (즉시): 구조 리스크 제거 + 신뢰도 기본 안전장치

### 1) `src/pipeline.py` 중복 정의 제거 (최우선)
- 현재 `pipeline.py`에 동일 함수가 반복 정의되어 있습니다.
- 파이썬은 마지막 정의가 유효하므로 당장 실행은 되더라도, 유지보수 시 "수정한 줄이 실제로 반영되지 않는" 장애를 유발하기 쉽습니다.

**왜 지금 해야 하나?**
- 운영 중 긴급 수정/핫픽스 실패 확률을 낮추는 가장 확실한 액션입니다.
- 모델 성능 개선보다 먼저, 코드 신뢰성을 확보해야 투자자 관점에서 리스크를 줄일 수 있습니다.

**완료 기준**
- 중복 함수 0건.
- 동일 동작 회귀 테스트 통과.

---

### 2) 백테스트 현실성 1차 강화 (슬리피지/유동성)
- 현재 백테스트는 고정 비용(bps) 기반으로 순수익을 계산합니다.
- 종목 유동성/변동성/체결 지연에 따른 비용 증가가 반영되지 않습니다.

**왜 지금 해야 하나?**
- 투자자의 실제 체감 성과와 백테스트 괴리를 줄이는 핵심입니다.
- 과도한 낙관 편향을 줄여 전략의 "실행 가능성"을 빠르게 검증할 수 있습니다.

**완료 기준**
- 동적 슬리피지(최소 2~3개 시나리오) 비교 리포트 추가.
- 비용 보수적으로 잡아도 양(+)의 성과 유지 여부 확인.

---

### 3) 시그널 출력에 리스크 경보 컬럼 추가
- 현재는 예측/신호 위주 출력이 강하고, 투자자가 즉시 위험을 판단할 경보형 정보가 부족합니다.

**추가 권장 컬럼**
- `risk_flag` (예: HIGH_UNCERTAINTY, LOW_LIQUIDITY, HIGH_TURNOVER)
- `position_size_hint` (신뢰도 기반 비중 힌트)

**왜 지금 해야 하나?**
- "무엇을 살까"보다 "얼마나 조심해야 하나"가 실전 투자에서 더 중요합니다.

---

## P1 (단기): 투자 의사결정 품질 개선

### 4) 확률 신뢰도 수치평가(ECE/Brier) 정식 편입
- 현재는 캘리브레이션 시각화가 존재하지만, 운영 판단용 수치 KPI가 부족합니다.
- ECE/Brier를 리포트 JSON에 포함해 모델 변경 시 품질 비교를 표준화해야 합니다.

**투자자 가치**
- "상승확률 70%" 숫자의 신뢰성을 수치로 검증 가능.

---

### 5) 포트폴리오 제약 추가 (편중/회전율 제한)
- 현재 Top-K 선별은 있으나 섹터/종목 편중 제어가 약합니다.
- turnover cap, 종목 최대 비중 cap, 최소 거래대금 필터를 추가해야 실전성과가 안정됩니다.

**투자자 가치**
- 급격한 스타일 쏠림, 매매 과잉(수수료 누수) 방지.

---

### 6) 투자자용 "종목 근거 카드" 생성
- 단순 점수 순위 대신, 종목별로
  - 핵심 기여 피처,
  - 최근 변동성 상태,
  - 신뢰도/불확실성,
  - 권장 비중 힌트
  를 카드 형태로 저장(이미지/CSV/JSON)하도록 확장.

**투자자 가치**
- 결과 신뢰와 설명 가능성을 동시에 확보.

---

## P2 (중기): 운영 자동화 및 드리프트 대응

### 7) 데이터/모델 드리프트 모니터링
- 입력 분포 변화, 예측 분포 왜곡, 실현 성과 하락을 자동 탐지해야 합니다.
- 경보 임계치 초과 시 재학습 또는 보수 모드 전환 규칙을 둡니다.

### 8) 실험/모델 레지스트리
- 데이터 버전, 파라미터, 모델 아티팩트, 성능을 묶어 추적하면
  "왜 성능이 변했는지"를 빠르게 역추적할 수 있습니다.

---

## 바로 실행 가능한 2주 액션 플랜

1. `pipeline.py` 중복 제거 + 회귀 테스트 강화.
2. 백테스트에 동적 슬리피지 옵션 추가(보수/중립/공격 시나리오).
3. 결과 CSV에 `risk_flag`, `position_size_hint` 추가.
4. 리포트 JSON에 ECE/Brier 추가.

위 4가지는 구현 난이도 대비 투자자 체감 효과가 가장 큽니다.
