> 목적: 티마 앱의 기능을 벤치마크해, **본 프로젝트 아키텍처와 정책 제약 안에서** 도입할 가치가 있는 업그레이드 후보를 우선순위로 정리한다.

## 0. 두 대상의 본질 차이 (비교 전 전제)

| 구분 | 티마(TIMA) | 본 프로젝트(stock_predict) |
|------|-----------|---------------------------|
| 형태 | 모바일 리테일 트레이딩 **앱(UI 제품)** | KOSPI200 **예측 파이프라인 + 카카오 챗봇(백엔드)** |
| 핵심 | 당일 주도 테마·대장주 빠른 파악 + 표준 매매 가이드(F존) | 다음날 `predicted_return`·상승확률·불확실성 산출 → top-k 백테스트 |
| 데이터 | 실시간 시세/호가/거래원/NXT/일정 | 일봉 OHLCV + 외부지표 + 투자자수급 + 공시/뉴스(표시용) |
| 출력 | 화면(테마/종목/F존/지수) + Push 알림 | `result/*.csv`, `pm_report.json`, 카카오 응답 |

> **따라서 모든 비교는 "UI 기능 복제"가 아니라 "본 프로젝트의 산출물·챗봇·리포트를 더 유용하게 만드는 데이터/로직 업그레이드" 관점으로 한다.**

### 절대 불변 정책 (업그레이드 시 반드시 준수)
- 매수/매도/관망 추천은 **오직 다음날 `predicted_return` 임계값**으로만 결정한다 (`src/domain/signal_policy.py:recommendation_from_signal`).
- 뉴스·공시·`news_impact_*`·테마/섹터 신호는 **표시·진단·정렬 보조 정보**이며 추천을 바꾸면 안 된다.
- 티마의 "F존" 같은 매매 가이드성 기능을 도입하더라도 **표시 전용 태그**로만 추가하고, "참고 지표일 뿐 투자조언이 아니다"라는 디스클레이머를 유지한다.

## 1. 기능 매핑: 티마 6대 영역 → 현재 프로젝트 대응 현황

| 티마 영역 | 핵심 기능 | 현재 프로젝트 대응 | 상태 |
|-----------|----------|-------------------|------|
| ① 마켓중심 | 테마 거래대금 합·상승률 정렬·**왕관(대장주)**·테마 뉴스 | `leader_confirmation_flag`(시장 **전체** 거래대금 상위 기준), `turnover_rank_daily` | ⚠️ 부분 — **테마/섹터 단위 집계·대장주는 없음** |
| ① 종목탭 | 시총·유통비율·2년내 상한가 횟수·관련테마·단기과열·호가·거래원·메모 | 외국인/기관 순매수, 공시/뉴스 컨텍스트, 가격·기술 피처 | ⚠️ 부분 — 상한가/단기과열/유통비율/메모 없음 |
| ② NXT | 대체거래소 종목 Pre/Aft 거래대금 | 입력에 `venue`/`session` 컬럼 인지하나 NXT 특화 없음, 시세원 없음 | ❌ 거의 없음 (데이터원 제약) |
| ③ 마켓일정 | 일/주/월 국내외 일정·경제지표 | DART **공시** 컨텍스트는 있으나 사전 **이벤트 캘린더** 없음 | ❌ 없음 |
| ④ F존 | F존+/SF존/골드존/38스윙 진입 가이드, Push 연동 | `signal_label`(strong_neg~strong_pos), `recommendation` | ⚠️ 부분 — 진입 구간 태그·Push 없음 |
| ⑤ 시장종합 | 국내외 지수 통합 | `external_features`가 KOSPI/KOSDAQ/S&P500/NASDAQ/SOX/VIX/USDKRW/UST10Y **수집** | ⚠️ 데이터는 있으나 **사용자용 요약 산출물 없음** |
| ⑥ 환경설정 | 알림내역·관심종목·Push·자동로그인·다크모드 | 챗봇은 pull-only, `news_impact` watchlist.csv만 존재 | ⚠️ 부분 — 사용자별 관심종목/Push 없음 |
| 공통 | 장전뉴스·특징주 뉴스 | `news_impact` 모듈(LLM 임팩트 스코어링, 표시용) | ⚠️ 자산은 있으나 다이제스트 산출물 없음 |

## 2. 업그레이드 후보 (우선순위별)

우선순위 기준: **가치(운영/예측 기여) × 아키텍처 적합도 ÷ 데이터·구현 난이도**. 정책 충돌 가능성도 명시한다.

### 🟢 P1 — 높은 가치 · 기존 자산 재활용 · 정책 안전

#### P1-A. 테마/섹터 단위 집계 + 섹터 대장주(왕관) 태깅
- **티마 기능**: 테마 거래대금 합, 상승률 순 정렬, 왕관(대장주) 식별 — 티마의 핵심 화면.
- **현재 상태**: `_leader_confirmation`(`src/features/investment_signals.py`)은 **시장 전체** 거래대금 상위 N의 동반 상승 여부만 본다. **종목→테마/섹터 매핑이 없어 "어느 테마가 주도하는지"는 산출하지 못한다.** 다만 `news_impact`에 `sector_keywords`·`company_master`·`company_relationships` 자산이 이미 있다.
- **제안**:
  1. 종목→섹터/테마 매핑 마스터 도입(news_impact의 `company_master`/`sector_keywords` 재활용 또는 KRX 업종 코드).
  2. 신규 `src/features/sector_features.py`: 일자별 **섹터 거래대금 합·평균 상승률·섹터 순위**, 섹터 내 거래대금/상승률 1위에 `theme_leader_flag`.
  3. `result_detail.csv`/`result_simple.csv`에 `sector`, `sector_turnover_rank`, `theme_leader_flag` 컬럼, `pm_report.json`에 "오늘 주도 섹터 Top-N" 추가.
- **정책 적합성**: ✅ 표시·진단·정렬용. `predicted_return` 추천은 불변(섹터 leader는 이미 `event_boost`에서 `signal_score`에만 반영되는 기존 패턴과 동일).
- **난이도**: 중 (매핑 데이터 확보가 관건, 로직은 groupby 집계).

#### P1-B. 시장종합 요약 산출물 (시장종합)
- **티마 기능**: 국내외 주요 지수 한 화면 통합.
- **현재 상태**: `src/features/external_features.py`가 이미 8개 지수/매크로를 수집하지만, **사용자가 볼 요약 산출물이 없다.**
- **제안**: 당일 지수 스냅샷(종가, 1일/5일 수익률, 20일 변동성)을 `result/market_overview.json`(+ CSV)으로 저장하고, 챗봇 `시장`/`지수` 발화에 응답. `pm_report.json`의 머리말에도 "시장 국면 요약" 추가.
- **정책 적합성**: ✅ 표시용.
- **난이도**: **낮음** — 데이터가 이미 파이프라인에 존재. 포맷팅·노출만 추가하면 됨. → **가성비 1순위.**

#### P1-C. 장전뉴스 / 특징주 뉴스 다이제스트 (공통)
- **티마 기능**: 장전뉴스, 특징주 뉴스 슬라이딩 탭.
- **현재 상태**: `news_impact` 모듈에 LLM 임팩트 스코어링·`ranking.py`가 있으나, "오늘의 상위 임팩트 뉴스" 다이제스트 산출물은 없다.
- **제안**: `news_impact` 랭킹 상위 N건을 `result/news_digest.json`(표시용)으로 정리, 챗봇 `뉴스`/`장전뉴스` 발화 응답. 종목 조회 응답에 해당 종목 상위 임팩트 뉴스 1~2건 첨부(이미 표시 전용 정책 하에 가능).
- **정책 적합성**: ✅ 표시 전용(예측·추천 불변). 기존 `news_impact_*` 표시 정책과 일치.
- **난이도**: 낮~중 — 기존 랭킹 재활용.

### 🟡 P2 — 중간 가치 · 추가 데이터/상태관리 필요

#### P2-D. 사용자별 관심종목 + 추천 변화 알림 (관심종목 + Push 알림)
- **티마 기능**: 관심종목 등록, F존/SF존/골드존/38스윙 Push 알림.
- **현재 상태**: 챗봇은 **pull-only**(사용자가 물어볼 때만 응답). 사용자별 관심종목 상태·알림 트리거 없음.
- **제안**:
  1. 챗봇에 관심종목 등록/조회 발화(예: `관심추가 005930`) + `result/runtime/watchlists/<user>.json` 저장.
  2. 신규 예측 실행 시 관심종목의 **추천 라벨 변화/시그널 임계 통과**를 감지해 Push.
  3. 채널: 카카오 알림톡/친구톡은 비즈니스 채널 필요 → 1차로는 "변화 큐"만 적재하고 사용자가 조회 시 노출, 2차로 채널 연동.
- **정책 적합성**: ✅ 알림 내용도 `predicted_return` 기반 추천만. 알림은 신호 자체를 바꾸지 않음.
- **난이도**: 중~높 (사용자 상태관리 + 카카오 비즈 채널 의존). Colab 환경 상태 영속화 이슈(`09_chatbot.md`의 P2 과제)와 함께 설계 필요.

#### P2-E. 상한가 / 단기과열 / 유통비율 플래그 (종목탭 확장)
- **티마 기능**: 상한가 표시, 단기과열 종목, 유통비율, 2년내 상한가 횟수.
- **현재 상태**: OHLCV로 **상한가(전일 대비 +29.x%)·단기과열(급등+거래량 급증)·N년 상한가 횟수**는 파생 가능하나 미구현. 유통비율은 외부 데이터 필요.
- **제안**: `src/features/price_features.py` 또는 `investment_signals.py`에 `upper_limit_flag`, `short_term_overheat_flag`, `upper_limit_count_2y` 추가(표시·진단용). 유통비율/시총은 마스터 데이터 확보 시 컬럼 추가.
- **정책 적합성**: ✅ 표시·진단용(이벤트 부스트로 `signal_score`에만 반영하는 기존 패턴 활용 가능, 추천 불변).
- **난이도**: 상한가/단기과열은 **낮음**(OHLCV 파생), 유통비율/시총은 데이터 의존.

#### P2-F. 마켓 일정 / 이벤트 캘린더 (마켓일정)
- **티마 기능**: 일/주/월 국내외 일정·경제지표.
- **현재 상태**: DART **공시**는 표시용으로 수집하나, 실적발표/배당락/경제지표 **사전 캘린더**는 없다.
- **제안**: 경제지표·실적·배당락 캘린더 수집 모듈(`src/data/` 신규) → `result/calendar.csv`, 근접 이벤트를 종목 컨텍스트/`pm_report`에 표시. FOMC·CPI 등 매크로 일정은 외부지표와 자연스럽게 연결.
- **정책 적합성**: ✅ 표시용.
- **난이도**: 중 (안정적 일정 데이터 소스 선정이 관건).

#### P2-G. F존 유사 "진입 구간" 표시 태그 (F존)
- **티마 기능**: F존+/SF존/골드존/38스윙 진입 구간 안내.
- **현재 상태**: `signal_label`(5단계 구간)·`recommendation`만 있고, 기술적 진입 패턴 태그는 없다. 단 RSI 풀백·52주 돌파/근접 등 신호 자산은 이미 `event_boost`에 존재.
- **제안**: 기존 신호를 조합해 **표시 전용** 진입 구간 태그(예: `눌림목_지지`, `52주_돌파`, `과매도_반등`)를 `entry_zone_tags`로 추가. 챗봇/리포트에 "참고 패턴"으로 노출.
- **정책 적합성**: ⚠️ **주의** — 매매 가이드처럼 오인되지 않도록 반드시 (1) `recommendation` 불변, (2) "참고 지표·투자조언 아님" 디스클레이머, (3) `predicted_return`과 분리 표기.
- **난이도**: 중 — 로직은 기존 피처 조합, 정책 검증 테스트 필수.

### 🔴 P3 — 데이터원 제약 크거나 본 프로젝트 범위 밖

#### P3-H. NXT(대체거래소) 종목 처리
- **티마 기능**: NXT Pre/Aft 거래대금, 종가대비.
- **현재 상태**: 입력에 `venue`/`session` 컬럼은 인지하나, `yfinance`는 NXT 시세를 제공하지 않음 → **데이터원 부재가 근본 제약.**
- **제안**: 우선순위 낮음. NXT 시세 소스를 확보할 경우에만 `venue`별 분리 표시 추가.

#### P3-I. 실시간 호가·프로그램 매매·거래원
- **티마 기능**: 호가창, 프로그램 매매, 거래원 정보.
- **현재 상태**: 일봉 기반 파이프라인이라 실시간 틱/호가 데이터 없음.
- **제안**: 실시간 데이터 인프라가 전제 → **현 아키텍처 범위 밖**. 도입 시 별도 서비스 설계 필요.

#### P3-J. UI 전용 기능 (다크모드·자동로그인·유튜브 버튼·메모장)
- 백엔드/챗봇 프로젝트에는 **비해당**. 단, "종목별 메모"는 사용자 상태관리(P2-D 인프라)가 생기면 챗봇 메모 발화로 소규모 도입 가능.

## 3. 우선순위 한눈에 보기

| 순위 | 후보 | 가치 | 난이도 | 정책 위험 | 비고 |
|------|------|------|--------|-----------|------|
| 1 | **P1-B 시장종합 요약** | 중상 | **낮음** | 없음 | 데이터 이미 존재 — 가성비 최고 |
| 2 | **P1-A 테마/섹터 + 대장주** | **높음** | 중 | 없음 | 티마 핵심, 매핑 데이터 확보 필요 |
| 3 | **P1-C 뉴스 다이제스트** | 중 | 낮~중 | 없음 | `news_impact` 재활용 |
| 4 | **P2-E 상한가/단기과열** | 중 | 낮음(파생) | 없음 | OHLCV 파생으로 빠름 |
| 5 | **P2-D 관심종목 + Push** | **높음** | 중~높 | 없음 | 카카오 비즈 채널·상태영속화 의존 |
| 6 | P2-F 마켓 캘린더 | 중 | 중 | 없음 | 데이터 소스 선정 관건 |
| 7 | P2-G F존 진입 태그 | 중 | 중 | ⚠️ 있음 | 디스클레이머·정책 테스트 필수 |
| 8 | P3-H/I/J | 낮음 | 높음 | — | 데이터원/범위 제약 |

## 5. 결론

티마의 강점은 **(1) 테마·대장주 중심의 시장 구조 파악**과 **(2) 표준화된 매매 가이드(F존)·Push 알림**이다. 본 프로젝트는 종목 단위 예측은 강하지만 **시장/테마 구조의 "위에서 내려다보는 뷰"와 사용자 능동 알림이 약하다.**

가장 효과적인 업그레이드는 **이미 보유한 데이터를 사용자용 산출물로 끌어올리는 것**(P1-B 시장종합, P1-C 뉴스 다이제스트)과, **새 매핑 데이터로 테마/섹터·대장주 뷰를 추가하는 것**(P1-A)이다. 매매 가이드(F존 유사)·Push는 가치가 높지만 각각 **정책 위험**과 **인프라 의존**이 있어 검토 후 단계적으로 진행한다.

> 모든 업그레이드는 "추천은 `predicted_return`만, 나머지는 표시·진단·정렬 보조"라는 본 프로젝트의 핵심 원칙을 깨지 않는 범위에서만 도입한다.
