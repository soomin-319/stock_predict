# 운영 가이드 단계별 개편 설계

## 목표

`docs/PROJECT_OPERATION_GUIDE.md`를 프로그램의 실제 실행 순서와 사용자가 챗봇에서 수행하는 작업을 중심으로 다시 구성한다. 처음 보는 운영자도 “무엇을 실행하고, 내부에서 어떤 일이 일어나며, 결과를 어디서 확인하는지” 순서대로 이해할 수 있어야 한다.

## 문서 독자

- 파이프라인을 처음 실행하는 운영자
- 결과 CSV와 JSON을 확인하는 사용자
- Kakao/Colab 챗봇에서 종목 조회, 결과 확인, 최신화, 추천 기능을 사용하는 사용자
- 세부 구현 파일을 찾아야 하는 개발자

## 개편 구조

문서 앞부분은 실행 흐름 중심으로 구성한다.

1. 프로그램 역할과 필수 원칙
2. 실행 전 준비
3. 파이프라인 실행 명령
4. 프로그램 내부 실행 단계
   - 설정 로드
   - 입력 데이터 준비
   - 정제와 유니버스 필터
   - 투자자·공시·뉴스 문맥 수집
   - 피처 생성
   - Walk-forward 검증과 OOF 예측
   - 신호 점수 계산과 백테스트
   - 최종 모델 학습과 최신 예측
   - 사용자용 결과 생성
   - 결과 파일 저장
5. 결과 확인 방법
6. 챗봇 시작과 초기화
7. 사용자 시나리오
   - 종목코드 입력: 예측 조회와 뉴스·공시 요약
   - `결과`: 진행 상태 또는 마지막 결과 확인
   - `최신화`: 선택 종목 예측과 요약 재실행
   - `추천`: 별도 실시간 추천 서비스 실행
   - `도움말`: 사용 가능한 입력 확인
8. 대기·실패 상황별 대응
9. 기술 상세와 모듈 책임
10. 테스트와 운영 점검표

## 사용자 시나리오 표현 원칙

- `요약`은 독립 명령으로 설명하지 않는다.
- 뉴스·공시 요약은 종목코드 조회 과정에서 필요할 때 생성되는 부가 정보로 설명한다.
- 요약과 뉴스 영향 정보는 `predicted_return`, 권고, 순위를 변경하지 않는다고 반복해서 명시한다.
- `추천`은 메인 파이프라인의 `predicted_return` 기반 권고와 구분한다. 챗봇의 `추천` 입력은 `RealTimeCloseBettingRecommendationService`를 호출하는 별도 사용자 흐름이다.
- 사용자가 입력한 문구, 프로그램 동작, 응답, 다음 행동을 표로 설명한다.

## 보존할 기술 내용

기존 문서의 피처, 모델, Walk-forward, OOF, 백테스트, 결과 파일, 모듈 책임 설명은 삭제하지 않는다. 실행 흐름을 먼저 이해한 뒤 참고할 수 있도록 문서 뒤쪽의 기술 상세로 재배치하거나 단계 설명에 연결한다.

## 정확성 검증

- `src/pipeline.py`의 진행 단계와 문서 순서를 비교한다.
- `src/chatbot/kakao_colab_bot.py`의 사용자 입력 처리 순서와 문서 시나리오를 비교한다.
- `추천`과 뉴스·공시 요약이 메인 예측 정책과 혼동되지 않는지 확인한다.
- Markdown 제목, 표, 명령 예시가 읽기 쉬운지 확인한다.
