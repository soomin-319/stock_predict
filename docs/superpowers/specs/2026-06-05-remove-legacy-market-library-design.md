# 미사용 시장 데이터 라이브러리 제거 설계

## 목표

저장소의 미사용 시장 데이터 라이브러리 코드, 테스트 가정, 운영 문서 언급을 모두 제거한다. 현재 호출되지 않는
KOSPI200 동적 로더는 삭제한다.

## 결정

- 기본 유니버스는 기존 `data/default_universe_kospi50_kosdaq50.csv`를 유지한다.
- 실시간 종가 베팅 기본 스캔은 기존대로 CSV의 KOSPI 50종목을 사용한다.
- 미사용 `_load_kospi200_symbols()`를 삭제한다.
- FDR은 추가하지 않는다. 삭제되는 로더가 호출되지 않고, FDR을 추가해도 현재 기능상 이점이 없다.
- 코드, 테스트, 활성 문서, 보관 문서에서 패키지별 문자열과 잘못된 통합 설명을 제거한다.

## 테스트

- 회귀 테스트에서 기본 서비스가 번들 유니버스를 사용하는지 검증한다.
- 저장소 추적 파일에 제거 대상 패키지 문자열이 남지 않았는지 검사한다.
- 관련 테스트와 전체 pytest를 실행한다.

## 비범위

- 투자자 수급 수집 기능 신규 구현
- 기본 유니버스를 KOSPI200으로 변경
- FDR 의존성 추가
